# 디자인 시스템 — Samsung.com 기반

samsung.com의 디자인 언어를 분석해 이 앱에 적용한 가이드.
핵심 인상: **프리미엄 · 미니멀 · 넓은 여백 · 화이트 베이스 · 블랙 강조 · 삼성 블루 포인트**. 장식보다 타이포·여백으로 위계를 만든다.

## 1. 컬러
| 토큰 | 값 | 용도 |
|------|-----|------|
| `--bg` | `#ffffff` | 기본 배경 |
| `--bg-soft` | `#f5f5f5` | 섹션 구분용 옅은 회색 배경 |
| `--ink` | `#0a0a0a` | 제목·핵심 텍스트, **주 CTA(블랙 버튼)** |
| `--ink-2` | `#545454` | 본문 보조 텍스트 |
| `--ink-3` | `#8a8a8a` | 캡션·플레이스홀더 |
| `--line` | `#dcdcdc` | 얇은 구분선/보더 |
| `--line-soft` | `#ececec` | 더 옅은 보더 |
| `--blue` | `#1428A0` | 삼성 브랜드 블루 — 링크·활성·포인트 |
| `--blue-bright` | `#2088ff` | 인터랙션·포커스 링 |
| `--blue-soft` | `#eef1fb` | 블루 약배경(선택/하이라이트) |

> 색은 **절제**가 핵심. 화면 대부분은 흑·백·회색, 블루는 포인트로만.

## 2. 타이포그래피 — Pretendard
- 폰트: **Pretendard** (system 폴백: Apple SD Gothic Neo, Malgun Gothic)
- 큰 제목: **800/700, 큰 사이즈, letter-spacing -0.02em** (타이트하게)
- 섹션 제목: 700, 18–22px
- 본문: 400/500, 14–15px, line-height 1.5
- 캡션: 12–13px, `--ink-3`

## 3. 버튼 (samsung 시그니처 = 블랙 pill)
- **Primary**: 배경 `--ink`(블랙), 흰 글자, **pill(완전 둥근) 또는 radius 24px**, padding 넉넉히(12×24), hover 시 살짝 밝게
- **Secondary**: 투명 배경 + `--ink` 1px 보더, pill
- **Text/Link**: `--blue`, 밑줄 없음, hover 밑줄
- 아이콘은 이모지 대신 **미니멀**하게(텍스트 위주), 필요한 곳만 단색 기호

## 4. 여백 & 레이아웃
- **넓은 여백**이 곧 프리미엄. 섹션 간 큰 수직 간격(28–40px)
- 콘텐츠 max-width 중앙 정렬(~1040px)
- 보더로 나누기보다 **배경 톤 차이(`--bg` vs `--bg-soft`)와 여백**으로 구분
- 카드: 흰 배경 + 아주 옅은 보더(`--line-soft`) 또는 미세 그림자, radius 16–20px

## 5. 컴포넌트 매핑(이 앱)
- 상단바: 흰 배경 + 하단 얇은 보더, 로고 블랙, 날짜/액션 미니멀
- 섹션 헤더: 블랙 굵은 제목 + 좌측 짧은 블루 바(선택) — 이모지 최소화
- Big3: 흰 카드 3개, 번호는 블루, 완료 시 블루 체크
- 타임라인 블록: **Big3 = 삼성 블루 솔리드(흰 글자)**, 일반 = 흰 카드 + 블루 좌측 보더 + 진한 글자, 미세 그림자
- 입력/텍스트영역: 옅은 보더, 포커스 시 블루 링
- 칩/필터: pill, 활성 = 블랙 배경 흰 글자
- 회고 통계: 큰 숫자(블랙), 차이는 블루

## 6. 적용 원칙
1. 색을 줄이고 **흑·백·여백**으로 고급감
2. 이모지 아이콘 최소화, **타이포 위계**로 정리
3. 포인트는 **삼성 블루 한 가지**로 통일
4. 폰트는 **Pretendard**로 깔끔하게

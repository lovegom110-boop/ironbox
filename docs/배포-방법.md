# 배포 방법 (IRONBOX)

> 한 줄 요약: **`main`에 git push 하면 Vercel이 알아서 웹앱을 배포합니다.**

---

## 1. 전체 구조 (헷갈리지 않게)

| 무엇 | 어디서 | 역할 |
|------|--------|------|
| **웹앱(화면·코드)** | **Vercel** | 실제 사이트를 띄우는 곳. push하면 자동 배포 |
| **데이터(할 일·노트 등)** | **Firebase (Firestore)** | DB만 사용. 로그인·동기화 저장소 |

- 주소: **https://ironbox-six.vercel.app/** (위젯은 **/widget**)
- 깃 저장소: github.com/lovegom110-boop/ironbox (브랜치 `main`)

> ⚠️ **"Firebase Hosting" 워크플로가 실패해도 정상입니다.**
> 우리는 Firebase를 **DB로만** 쓰고, 화면은 Vercel이 띄워요. 그래서
> `.github/workflows/firebase-hosting.yml`(Firebase Hosting 배포)은 안 쓰는 호스팅을
> 배포하려다 **매번 실패**하지만 **앱 동작엔 전혀 영향 없습니다.** 무시하면 됩니다.

---

## 2. 배포하기 (기본)

코드를 고친 뒤:

```bash
git add <고친 파일들>
git commit -m "무엇을 왜 고쳤는지"
git push origin main      # ← 이 순간 Vercel이 자동으로 새 배포 시작
```

- push 후 **약 30초~1분**이면 라이브에 반영됩니다.
- 별도 "배포 버튼"을 누를 필요 없음. push = 배포.

### JS/CSS를 고쳤다면 — 캐시 버전 올리기
PWA(설치형)·서비스워커가 옛 파일을 계속 쓰지 않도록, JS/CSS를 바꿨으면
[`sw.js`](../sw.js) 맨 위 캐시 번호를 **+1** 합니다.

```js
const CACHE = "elon-diary-v30";   // ← 숫자를 올림 (예: v30 → v31)
```

이러면 사용자가 다음에 열 때 새 버전을 자동으로 받습니다.

## 3. 배포 성공 확인하는 법

### 방법 A — 그냥 사이트 열어보기 (제일 쉬움)
**https://ironbox-six.vercel.app/** 를 열고 새로고침(Ctrl+Shift+R) 해서 바뀐 게 보이면 끝.

### 방법 B — GitHub에서 Vercel 상태 확인
방금 push한 커밋 옆에 **Vercel** 체크가 초록색(✓ "Deployment has completed")이면 성공.
- 커밋 목록: github.com/lovegom110-boop/ironbox/commits/main

> 💡 **회사망에서는 `*.vercel.app` 접속이 막혀** 있어요(사이트가 안 열림 = 죽은 게 아니라 망 차단).
> 회사 PC에서 확인이 안 되면 **휴대폰(LTE/5G)이나 집**에서 열어보거나, 위 방법 B(GitHub 체크)로 확인하세요.

## 4. 막힐 때

| 증상 | 확인 / 해결 |
|------|-------------|
| 바뀐 게 안 보임 | 새로고침(Ctrl+Shift+R). PWA면 `sw.js` 캐시 번호 올렸는지 확인 |
| 회사 PC에서 사이트가 안 열림 | `*.vercel.app` 망 차단일 가능성 → 폰/집에서 열어보기 |
| Firebase Hosting 워크플로 빨간 실패 | **정상**(안 쓰는 호스팅). 무시. (1번 참고) |
| Vercel 체크가 빨간 실패 | 그때만 진짜 배포 실패 → 커밋의 Vercel 링크에서 빌드 로그 확인 |
| 데이터가 안 보임/저장 안 됨 | 로그인 여부·Firestore(다른 문제). 배포와 별개 |

## 5. 참고 (데이터·보안)

- Firestore 보안규칙: [`firestore.rules`](../firestore.rules) — `users/{uid}` 아래는 **본인만** 읽기/쓰기.
- 규칙 배포는 Firebase Hosting 워크플로 안에서 **best-effort**로 같이 시도하지만,
  서비스계정 권한 문제로 실패할 수 있음(앱엔 영향 없음). 필요 시 Firebase 콘솔에서 수동 반영.

# 노트장 — 인쇄 / PDF (보이는 서식 그대로) — 설계

- 날짜: 2026-06-19
- 대상: IRONBOX (일론 다이어리)
- 관련: [노트장 설계](2026-06-07-notebook-design.md). 이번 변경은 노트장 편집 화면에 **인쇄/PDF 내보내기**를 더하는 작은 기능.

## 배경 / 해결하려는 문제
노트장은 마크다운으로 저장하고 화면에선 Toast UI 에디터로 **서식 있는 모습(WYSIWYG)** 으로 보입니다.
하지만 지금까지 밖으로 빼는 길은 **`.md` 파일 다운로드**와 **마크다운 원문 보기/복사** 뿐이었습니다.
즉 **화면에 보이는 서식 그대로(제목·목록·표 등)** 종이로 인쇄하거나 PDF로 저장하는 방법이 **아예 없었습니다.**

## 목표
- 노트 편집 화면에서 버튼 한 번으로 **보이는 서식 그대로** 인쇄.
- 같은 버튼(인쇄창)에서 **"PDF로 저장"** 으로 PDF도 받기.
- 추가 라이브러리 없이(오프라인 PWA·회사망 차단 환경에서 안전하게).

## 범위
**할 것**: 상단 「노트」로 들어가는 **노트장(학습 노트)** 의 개별 노트 인쇄/PDF.
**안 할 것**: 하루 다이어리·「오늘의 메모」 포스트잇은 이번 범위 밖(요청에 따라 노트장만).
전용 PDF 생성 라이브러리(jsPDF/html2pdf) 도입 안 함.

## 핵심 결정과 근거

### 1) "인쇄 전용 영역 + 인쇄용 스타일" 방식 (채택)
- 노트 내용을 화면 속 **숨은 `#nb-print` 영역**에 다시 그려 넣고, `@media print`로 **인쇄할 때만 이 영역만** 흰 종이로 출력합니다(앱 화면·툴바·어두운 배경은 전부 숨김).
- **왜**: 이 앱은 오프라인 PWA라 외부 CDN을 못 받습니다. 이 방식은 **앱이 이미 미리보기·검색에 쓰는 안전 렌더(`Notes.mdToSafeHtml` = marked→DOMPurify)** 를 그대로 재사용 → 출력 서식이 **화면과 일치**하고, 새 의존성이 **0개**입니다.
- **검토했지만 탈락한 대안**
  - *새 창 열어 인쇄(`window.open`)*: 설치형 PWA(standalone)에선 새 창이 막히거나 어색하게 떠 **불안정**.
  - *전용 PDF 라이브러리*: 용량↑, 한글 폰트·표 깨짐 위험, 오프라인 번들 부담. 사용자도 "브라우저 인쇄→PDF"를 선택.

### 2) PDF는 "브라우저 인쇄창 → PDF로 저장"
- 별도 PDF 버튼을 두지 않고, **인쇄 버튼 하나가 인쇄와 PDF를 동시에** 해결합니다(인쇄창에서 프린터 대신 "PDF로 저장" 선택).
- PDF 저장 시 **기본 파일명이 노트 제목**으로 뜨도록, 인쇄 직전 `document.title`을 잠깐 노트 제목으로 바꾸고 인쇄가 끝나면(`afterprint`) 원래대로 되돌립니다.

### 3) 흰 종이 + 검정 글자 (다크 그대로 X)
- "화면 그대로"는 **마크다운 원문이 아니라 보이는 서식 그대로**를 뜻한다고 확정.
- 종이엔 **흰 배경·검정 글자**가 읽기 좋고 잉크를 아낍니다. 표는 회색 헤더·얇은 테두리, 코드블록은 옅은 회색 박스 등 화면 톤(Samsung 라이트)을 종이용으로 정돈.

### 4) 제목 + 작은 날짜줄 포함
- 인쇄물 맨 위에 **노트 제목(H1)** 과 그 아래 **작은 회색 한 줄(생성/수정 일시)** 을 둡니다. 둘 다 화면에도 보이는 정보라 "화면 그대로"에 부합하고, 출력물 식별에 도움.

## 동작 (데이터 흐름)
1. 편집 화면 우하단 **`🖨 인쇄 / PDF`** 버튼 클릭 → `printNote(n)` 실행.
2. 현재 노트의 마크다운(`currentMarkdown` = 편집 중 내용 우선) → `Notes.mdToSafeHtml`로 **안전 HTML** 변환.
3. `#nb-print`에 제목·날짜줄·본문을 채우고, 본문 내 링크는 새 탭(`rel=noopener`)으로.
4. `document.title`을 노트 제목으로 잠깐 변경 → `window.print()`.
5. 인쇄/취소 후 `afterprint`에서 제목 복구 + `#nb-print` 비움(원상복구).

## 인쇄 레이아웃 규칙 (페이지 깨짐 방지)
- 제목(H1~H6) 뒤에서 페이지가 끊기지 않게(`break-after:avoid`).
- 표의 행(`tr`)·인용문·코드블록·이미지는 가능한 한 페이지 경계에서 **쪼개지지 않게**(`break-inside:avoid`).
- 코드블록은 줄바꿈 허용(`white-space:pre-wrap`)으로 가로 넘침 방지. 이미지 `max-width:100%`.
- 종이 여백 `@page{margin:18mm 16mm}`.

## 신규/영향 파일
- `js/notebook.js` — `printNote(n)` 추가 + 우하단 액션에 `🖨 인쇄 / PDF` 버튼.
- `index.html` — `<body>` 직속에 빈 `<div id="nb-print">` 한 줄(인쇄 전용 영역).
- `css/style.css` — `#nb-print{display:none}` + `@media print{…}` 블록(흰 종이 출력 스타일). 액션 버튼 줄 `flex-wrap`로 3개 버튼 줄바꿈 허용.
- `sw.js` — 캐시 `v43 → v44`(수정된 셸 파일이 사용자에게 새로 받아지도록).
- `test/notebook-deploy.test.js` — 인쇄 기능 회귀 검증 추가(`printNote`/`window.print`/`mdToSafeHtml`/`afterprint`/`#nb-print`/`@media print`).

## 알려진 한계 / 참고
- 출력은 Toast UI 에디터 픽셀을 그대로 복사하는 게 아니라 **같은 렌더러로 다시 그린 것** → 서식·내용은 화면과 동일하나 글꼴·간격이 1px까지 같지는 않음(종이엔 오히려 더 정갈).
- `afterprint`를 지원하지 않는 아주 오래된 브라우저에선 제목 복구가 늦을 수 있으나 동작엔 영향 없음.

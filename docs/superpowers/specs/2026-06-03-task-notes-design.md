# 작업별 세부 메모 (Task Notes) — 설계

- 날짜: 2026-06-03
- 대상: IRONBOX (바닐라 HTML/CSS/JS PWA 타임박스 다이어리)
- 배경: TickTick 벤치마킹 결과, 작업(할일)별 상세 노트가 빠른 차용 포인트로 도출됨.

## 목표
각 할 일에 자유로운 **세부 메모(멀티라인)** 를 달고, 인라인으로 펼쳐 편집/저장한다.

## 데이터 모델
- `store.js`의 `newTask()`에 `note: ""` 필드 추가.
- 마이그레이션 불필요: `normalizeDay()`의 `Object.assign(newTask(""), t)`가 기존 작업에 `note:""`를 자동 보강(읽기 경로 정규화는 직전 버그수정에서 이미 적용됨).

## UI / 상호작용 (인라인 펼침)
- 할 일 줄에 **📝 메모 버튼** 추가(별표와 수정 사이). 메모가 있으면 `on` 상태(채워진 느낌), 없으면 흐리게 표시해 유무를 한눈에.
- 클릭 → 그 줄 아래 **textarea 펼침**(멀티라인, 길이 제한 없음). 같은 버튼 재클릭 또는 textarea 바깥 클릭(blur) 시 접힘.
- 펼침 상태는 `state.openNoteId`로 1개만 추적. 메모 편집 중에는 mid-edit 가드(`renderTasks`에서 `[data-note-open]` 살아있으면 목록 재구축 보류)로 입력 소실 방지. (직전에 넣은 인라인 편집 가드와 동일 패턴, 별도 키)
- 저장: textarea `input` 시 `t.note` 갱신 + `saveDebounced`(400ms) → Firestore. 제목 인라인 편집과 토글이 충돌하지 않게 분리.

## 검색 포함 (결정 ①: ON)
- `store.js`의 `search()` 작업 필터에 `note` 포함: `t.text||t.note`에 질의어가 있으면 매치. (결과 표시는 기존 작업 줄 그대로)

## 내보내기 포함 (결정 ②: ON)
- 업무일지 텍스트(.txt): 각 배치 작업 줄 아래에 메모를 들여쓰기로 출력(메모 있을 때만).
- Google 캘린더: 이벤트 `description`에 `#태그` + 메모를 함께 기록(빈 줄로 구분).

## 타임라인 표시 (결정 ③: ON)
- `timebox.js` 블록에 메모가 있으면 작은 **📝** 표시(표시 전용, 클릭 동작 없음).

## 영향 파일
- `js/store.js` — `note` 필드, `search()` 포함
- `js/app.js` — 📝 버튼·인라인 펼침·저장·`openNoteId` 가드·텍스트 내보내기
- `js/timebox.js` — 블록 📝 표시
- `js/gcal.js` — `description`에 메모 추가
- `css/style.css` — 메모 버튼·펼침 패널 스타일
- `sw.js` — 캐시 `v12 → v13`
- `CLAUDE.md` — 변경 기록 한 줄

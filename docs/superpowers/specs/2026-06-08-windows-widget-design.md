# IRONBOX 윈도우 바탕화면 위젯 — 설계

> 작성 2026-06-08. 브레인스토밍 합의 내용 기록.

## 한 줄 요약
지금 IRONBOX 웹앱 코드를 재활용해, **오늘 할 일만 가볍게 보여주는 작은 창**을 바탕화면에
항상 떠 있게 띄운다. 같은 구글 계정·같은 클라우드 데이터를 공유한다.

## 확정된 결정
- **형태**: 바탕화면에 항상 위(always-on-top)로 떠 있는 작은 프레임리스 창.
- **범위**: 오늘 날짜의 **Big 3 + 할 일 목록만**. 시간 배치(타임박스)·달력·회고는 위젯에 없음(전체 앱에서).
- **기능**: 보기 + 완료 체크 + 빠른 추가 + "자세히 보기"(전체 앱 열기).
- **데이터**: 웹앱과 동일한 Firestore(`users/{uid}/days`)를 **같은 구글 계정**으로 공유.
  한쪽 변경이 다른 쪽에 실시간 반영(웹 ↔ 위젯 ↔ 폰).
- **제작 방식**: Tauri(가벼움 — 윈도우 내장 WebView2 사용, 앱 용량/메모리 작음).
- **코드 구조**: 기존 프로젝트 안에 위젯 전용 화면 `widget.html` + `js/widget.js` + 위젯 CSS를 추가.
  데이터/로그인/클라우드는 기존 `store.js`·`auth.js`·`firebase-*`를 그대로 재사용("한 살림").
- **톤**: 기존과 동일(화이트 + 삼성 블루 `#1428A0` + Pretendard).

## 데이터 매핑(기존 재사용)
- 오늘 키: 기존 앱과 동일한 `YYYY-MM-DD`(KST) 규칙.
- Big 3 = `day.tasks` 중 ★(star/big3) 표시 항목, 할 일 = 나머지 `day.tasks`.
- 완료 체크 = task의 `done` 토글 후 `Store.saveDay`.
- 빠른 추가 = 기존 task 추가 경로와 동일한 형태로 `day.tasks`에 push 후 저장.
- 실시간 동기화 = Firestore `onSnapshot`(기존 init 재사용).

## 두 가지 현실 주의점(알고 시작)
1. **위젯 첫 로그인**: 작은 창 안에서 구글 로그인 붙이는 게 제일 까다로운 지점.
   기본 방향 = 처음 한 번 로그인 → Firebase 세션이 유지되어 이후 자동.
   WebView2에서 팝업 로그인이 막히면 시스템 브라우저 OAuth → 딥링크로 세션 이어받기로 대체.
2. **굽기(build) 시점 인터넷**: Tauri 빌드 때 도구를 내려받음. 회사망에서 막힐 수 있어
   인터넷 자유로운 환경(집 PC 등)에서 빌드 권장. 다 구운 뒤 실행은 오프라인 OK.

## 구현 순서(계획 메모)
1. `widget.html` + `js/widget.js`(+CSS): 브라우저에서 바로 동작하는 위젯 화면(보기·체크·추가·자세히보기).
   → 기존 dev 서버(localhost)에서 즉시 확인 가능.
2. Tauri 셸: 위 페이지를 가리키는 작은 always-on-top·프레임리스 창 설정 + 위치/크기 기억.
3. 로그인 연결 마무리 + "윈도우 시작 시 자동 실행"(선택).
